我们注意到,代码中 movieCode 是 String 类型的,而我们更新表的时候却没加单引号或双引号，而 Mysq 对这种隐式类型转化会将前置 0 给删除掉。导致了无法查询到数据的 bug.

这里有两点需要说明

1.为什么 update 的时候可以匹配到数据?

如果我们写这样的 sql 语句.

假设这里的列是 varchar 类型的, 那么 sql 执行查询时 会将 where 条件后的前缀 0 都去掉

  • 00001000 会变为 1000
  • 0012300 会变为 12300

即 无论前缀有多少个 0 都会进行删除. 然后匹配的时候 也会对已有的 varchar 类型做一个去前缀 0 的操作。
假设 where 条件 column = 012340 隐式转为 ‘12340’. 表里假如有 cloumn = ‘0012340’, column = ‘00012340’都将与该条件匹配.

隐式类型转换规则

  • “000123 你好 abc123” 转 数字会变为 123
  • “000123” 会变为 123
  • “123abc 你好” 会变为 123
  • “你好李焕英” 会变为 0
  • 即从左到右 删掉前置 0，到第一个非数值型字符为止。如果最左侧直接是非数值型 则直接转为 0
